# Ручной чек-лист приёмки UI (Task 7)

Сценарий из технического плана (Task 7, Step 1). Запуск: `npm run dev`,
открыть локальный URL, который выведет Vite.

- [ ] Открыть приложение: видна пустая доска 9×9, статус «Фаза: игра. Ходят чёрные.», кнопка «Пас».
- [ ] Кликнуть пустое пересечение.
- [ ] Увидеть камень и смену хода: появился чёрный камень, статус сообщает «Ходят белые».
- [ ] Увидеть объяснение недопустимого хода: кликнуть уже занятое пересечение — рядом с доской появляется текст «Недопустимый ход: это пересечение уже занято.», позиция и очередь хода не меняются.
- [ ] Выполнить захват: окружить одиночный камень противника со всех четырёх сторон (например, белый камень в углу окружается двумя чёрными) — захваченный камень исчезает с доски.
- [ ] Сделать два паса: после первого паса очередь переходит и появляется подсказка о втором пасе; после второго подряд игра переходит в фазу «отметка мёртвых групп», кнопка «Пас» исчезает, появляется кнопка «Игроки согласовали счёт».
- [ ] Отметить группу мёртвой: клик по камню отмечает всю его связную группу (пунктирная рамка и крест «✕», в подписи пересечения — «группа отмечена мёртвой»); повторный клик снимает отметку; клики по пустым пересечениям в этой фазе недоступны.
- [ ] Подтвердить счёт и увидеть победителя/итоговые очки: после нажатия «Игроки согласовали счёт» показываются очки чёрных (камни и территория), очки белых, коми белых 7,5, итог белых с коми и победитель; доска становится неактивной.

> **Примечание.** Мёртвые группы согласовывают только люди за общим
> экраном: приложение не определяет статус групп само, оно лишь
> фиксирует отметки, о которых оба игрока договорились вслух, до нажатия
> кнопки «Игроки согласовали счёт».
