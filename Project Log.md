# Project Log — Игра Go

## 2026-07-16 — Инициализация

**Решение.** Создан новый проект о настольной игре го (бадук).

**Подтверждённый scope.** Единственное утверждённое требование на этом этапе — предмет проекта: настольная игра го. Платформа, движок, игровые режимы, правила, визуальный стиль, модель ИИ, монетизация и интеграции ещё не определены.

**Состояние.** Созданы только стартовая папка и документация. Код, зависимости, Git-репозиторий, внешние сервисы и публикации не создавались.

**Следующее действие.** Согласовать короткий игровой бриф и границы первой версии.

## 2026-07-16 — Выбор первого MVP

**Решение.** Первая версия — web-игра го на доске 9×9 для двух игроков за одним экраном.

**Не входит в MVP.** ИИ, онлайн-матчи, аккаунты, рейтинги, облачное хранение и внешние сервисы.

**Открытые правила.** До реализации нужно выбрать набор правил, обработку ко/самоубийства, условия завершения и подсчёт результата. Подробности вынесены в `Game Brief.md`.

**Следующее действие.** Выбрать правила и модель подсчёта для первой playable-версии.

## 2026-07-16 — Набор правил MVP

**Решение.** Для web-MVP зафиксированы Chinese area rules: окончание после двух последовательных пасов; результат складывается из живых камней и окружённой территории.

**Открытые параметры.** Формальная версия правил, обработка повторов позиции/ко, самоубийства, мёртвых групп, значение коми и UI перехода к подсчёту ещё не выбраны.

**Следующее действие.** Утвердить эти параметры, затем подготовить технический план и тестовые сценарии движка.

## 2026-07-16 — Формализация правил MVP

**Решение.** Для MVP принят рабочий набор Chinese area rules по переводу *Rules of Weiqi (Go), 2002*: недопустимый ход без свобод без взятия, запрет повторов полной позиции, два последовательных паса, согласование мёртвых групп и area scoring.

**Баланс.** Белым добавляется коми 7,5.

**Граница.** Это рабочая спецификация продукта; первичным источником правил остаётся китайский оригинал, на который ссылается используемый английский перевод.

**Следующее действие.** Подготовить технический план web-MVP и тестовые сценарии движка до начала кода.

## 2026-07-17 — Реализация и верификация web-MVP

**Решение.** Web-MVP реализован: доска 9×9, два игрока за одним экраном, Chinese area rules, взятия, отклонение самоубийства и повтора полной позиции, два последовательных паса, совместная разметка мёртвых групп, area scoring с коми 7,5 для белых, доступный русскоязычный браузерный интерфейс.

**Верификация (2026-07-17T03:33:12+05:00).** `npm test -- --run` — пройдено: 6 файлов, 113 тестов; `npm run typecheck` — пройдено; `npm run build` — пройдено, 13 модулей трансформировано. Повторный прогон тех же проверок позднее также прошёл.

**Браузерная проверка.** Фактически проверены: стартовый экран; легальный ход и смена очереди; отклонение хода в занятый пункт; два паса и переход к разметке мёртвых групп; разметка мёртвых; подтверждение подсчёта и результат. Сценарий взятия в браузере вручную не прогонялся — регрессия взятий покрыта автотестами.

**Границы.** Зависимости — только TypeScript, Vite, Vitest; runtime/UI-фреймворков нет. Git-репозиторий не создавался; сеть, ИИ, аккаунты, аналитика и постоянное хранение не добавлялись. Публикаций, деплоя и релиза не было.

**Следующее действие.** Пройти ручную приёмку по `docs/manual-checklist.md` (запуск: `npm run dev`).
